The steps of contributing to any open-source project on GitHub is -

  1. Find a project to contribute to

    2. There are millions of projects on GitHub, so we should be able to find one interests that we have the skills to contribute to. We can search for projects by keyword,language or topic.

  2. Read the project's contributing guidelines

    Most projects have a contribution guidelines document that explains how to contribute to the project.This document will tell what kind of contributions are,how to format our code and how to submit our changes.

  3. Create a fork of the project

    A fork is a copy of the project that we can make chnages to without affecting the main codebase. To create a fork, click the Fork button on the project's page.

  4. Clone the fork to our local computer

    Once we have forked the project, we need to clone it to our local computer. This will create a copy of project on our hard drive which we can work on.

  5. Make our changes

    Make the changes to the code that we want to contribute. Be sure to follow the project's contribution guidelines and to test our changes thoroughly.

  6. Commit our changes

    Once we are done the chnages, commit them to our local repository.This will create a snapshot of the code at that point in time.

  7. Push our changes to our fork

    Once we have commited our chnages,push them to our fork on GitHub. This will make changes avilable to other developers.

  8. Create a pull request

    A pull request is a way to request that our changes be merged into the main codebase. To create a pull request,click the pull Requests button on our fork's page.

  9. Wait for our pull request to be reviewed

    Other developers will review our changes and provide feedback.Once our changes are approved, they will be merged into the main codebase.
